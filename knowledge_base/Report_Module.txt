# Keywords
reports module, shipment reports, tracking report, invoice report, freight compare, error log, logistics reporting

# Reports Module Overview

The Reports module is used to monitor shipment activity, billing, tracking, and system errors.

It includes the following sections:
- End Of Day
- Ship Tracking
- Daily Shipment
- Re-Print Label
- Payment List
- Generate Invoice
- Freight Compare
- Error Log

This module helps in operational monitoring and financial tracking.

# End Of Day

The End Of Day page is used to finalize daily shipments for closure and billing.

## Filters
- Service Provider
- Carrier
- Date
- Account / Meter

## Information Displayed
- Tracking Number
- Order ID
- Service Provider
- Carrier
- Service Description
- Status
- Weight (KG)
- Shipping Charges (INR)
- Tracking Barcode

## Actions
- Select shipments
- Closure Meter → finalize shipments
- View Closure Data → view summary

## Purpose
Used to close daily shipment records for billing and reporting.

# Freight Compare

Used to compare expected vs actual shipping charges.

## Filters
- Service Provider
- Carrier
- Select Days

## Information Displayed
- Order ID
- Service Provider
- Tracking Number
- Carrier
- Date
- Weight
- Shipping Charges
- Actual Charge
- Difference

## Logic

If difference is within ±10% → Matched  
If difference > ±10% → Mismatch  

## Example
Shipping: 1000  
Actual: 950 → Matched  
Actual: 1200 → Mismatch  

## Purpose
Used to detect billing discrepancies.
